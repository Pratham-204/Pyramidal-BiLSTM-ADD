# Audio Deepfake Detection using Pyramidal BiLSTM 🎙️🕵️‍♂️

Detecting highly realistic synthetic speech generated by modern Text-to-Speech (TTS) and Voice Conversion (VC) models. 

This repository contains the implementation of an end-to-end audio deepfake detection system that utilizes a **Pyramidal Bi-directional LSTM (P-BiLSTM)**. By moving beyond traditional acoustic features (like MFCCs) and leveraging high-resolution Mel-spectrograms, this model efficiently captures both fine-grained digital artifacts and long-term structural anomalies in synthetic speech.

## 🚀 Key Features

* **Pyramidal Architecture:** Implements a hierarchical temporal compression mechanism (Time-Distributed Dense layers) that significantly reduces computational load for long audio sequences without losing context.
* **High-Resolution Spectral Analysis:** Utilizes 80-bin Mel-spectrograms to capture the specific high-frequency "mechanical" signatures and phase irregularities left by neural vocoders.
* **Computational Efficiency:** Achieves linear time complexity $O(T)$, offering a lightweight and scalable alternative to parameter-heavy Transformer models for real-world deployment.

## 📊 Datasets & Performance

The model was trained and evaluated on two major benchmarks:

* **Fake-or-Real (FoR) Dataset:** * **99.25% Accuracy** on clean, full-length utterances (`for-norm`).
  * Demonstrated strong resilience and feature extraction capabilities on truncated 2-second audio clips.
* **ASVspoof 2019 LA Dataset:** * Achieved an Equal Error Rate (EER) of 0.1053 and an AUC-ROC of 0.9608.

## 🧠 Model Architecture

1. **Input:** High-resolution 80-bin Mel-spectrograms.
2. **Bottom BiLSTM Layer:** Scans raw frames forward and backward to detect localized, micro-level generative glitches.
3. **Temporal Compression (Pyramid Step):** Merges sequential timesteps to halve the sequence length, simulating a broader contextual window.
4. **Top BiLSTM Layer:** Analyzes the compressed sequence to identify macroscopic unnatural prosody and rhythmic inconsistencies.
5. **Attention & Classification:** An attention mechanism weighs critical frames before a final Dense/Sigmoid layer outputs a 'Bona Fide' (0) or 'Spoof' (1) probability.

## Setup & Installation

1. **Clone the repository:**
   ```bash
   git clone <your-repository-url>
   cd btp
   ```

2. **Create a virtual environment:**
   ```bash
   python -m venv venv1
   venv1\Scripts\activate
   ```

3. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

## Usage

Place your dataset inside the project directory (e.g., `for-rerec/` or `ASVspoof/`) and adjust the paths in `main.py`.

Run the training pipeline:
```bash
python main.py
```

### Feature Caching
To speed up subsequent training runs, extracted Mel-Spectrograms are automatically cached to disk. If you change your augmentation parameters, force a fresh extraction by running:
```bash
python main.py --fresh-features
```

## Logging
Logs are printed to the console and automatically saved to the `logs/` directory.
